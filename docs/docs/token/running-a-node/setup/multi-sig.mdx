This tutorial walks through the process of delegating 100 $AUDIO from a freshly created [Safe{Wallet}](https://safe.global/) ("Safe") to a node operated by the Open Audio Foundation.

:::note

Nodes operated by the Open Audio Foundation have a 100% fee associated with them - 3rd-parties should not choose to delegate to the Open Audio Foundation nodes, and this fee is set as such to discourage that.

<!-- TODO: link to "read more about fees and delegation here" docs -->

:::

- `0x0x1cF73c5023572F2d5dc6BD3c5E4F24b4F3b6B76F`

### Add Owners and Set Signing Thresholds

Add multiple trusted wallet addresses as signers and confirm the required minimum number of signers to accommodate your security needs.


<figure className="screensnippet-wrapper">
  <img src={useBaseUrl('/img/multisig-wallet/create/create-signers.png')}  />
  <figcaption>
      Giving each member wallet address a meaningful nickname helps trouble shoot and identify wallets in the future.
  </figcaption>
</figure>


Work with the other members of the Safe to determine the safety and convenience trade offs and select a minimum number of member wallets required to sign transactions. More signers can imply greater security, but it also requires more signers to actively participate in the process. On the other hand, a threshold of 1 could be chosen to allow any member wallet to act on behalf of the group.



<figure className="screensnippet-wrapper">
  <img src={useBaseUrl('/img/multisig-wallet/create/create-threshold.png')}  />
  <figcaption>
      This tutorial requires 3 of the 5 signers to participate in signing transactions, meaning that any subset of 3 wallet members will be required to sign transactions.
  </figcaption>
</figure>

## Function Calls

You now need to call two functions via the safe - approve (to permit the delegation system to
transfer $AUDIO tokens on your behalf) and delegation


### Approve

Select the "New Transaction" button on the Safe home page, and then choose "Transaction Builder".

<figure className="screensnippet-wrapper">
  <img src={useBaseUrl('/img/multisig-wallet/transact/transact-new.png')}  />

  </figcaption>
</figure>

<figure className="screensnippet-wrapper">

  <figcaption>
      Builder button
  </figcaption>
</figure>


On the next screen, complete the form as follows:

- Address or ENS Name: use the [AUDIO address `0x18aaa7115705e8be94bffebde57af9bfc265b998`](https://etherscan.io/token/0x18aaa7115705e8be94bffebde57af9bfc265b998?a=0x44617f9dced9787c3b06a05b35b4c779a2aa1334)
- Enter ABI: copy and paste the ABI code from the [bottom of this page](/token/multi-sig#audius-token), or directly from source on [GitHub](https://github.com/AudiusProject/audius-protocol/blob/7b53f915cd2b5581e877ed66ef050825d47e17e1/packages/libs/src/eth-contracts/ABIs/AudiusToken.json)
- Contract Method Selector: `addMinter`
- account (address): use the staking proxy address `0xe6D97B2099F142513be7A2a068bE040656Ae4591`

- Fill in the modal as follows:

  - Audius token contract address should be the address:
    `0x18aaa7115705e8be94bffebde57af9bfc265b998`
  - The ABI it detects will be incorrect - paste the long data from above for `Audius Token` into
    the ABI field
    <!-- TODO: link to json blob at bottom of page -->
  - You should now see a list of functions available underneath the ABI field like this when you
    click the dropdown:

  <figure className="screensnippet-wrapper">
    <img src={useBaseUrl('/img/gnosis-safe/function-list.png')} />
    <figcaption>caption goes here</figcaption>
  </figure>

  - Select "Approve"
  - Put in the Staking proxy address as the “spender”: `0xe6D97B2099F142513be7A2a068bE040656Ae4591`
  - Amount is taken in wei rather than full $AUDIO tokens - please convert the number of $AUDIO
    tokens you want to delegate (in this tutorial 100) to the equivalent wei number by multiplying
    by “1,000,000,000,000,000,000” (10^18). (eg. see here for the tutorial calculation:
    https://www.wolframalpha.com/input/?i=1%2C000%2C000%2C000%2C000%2C000%2C000*100). Put this
    resulting number in as “amount” (in our case 100000000000000000000).
    - To validate this is being done correctly, we recommend double-checking your number using an
      ETH to WEI converter such as this: [https://eth-converter.com/](https://eth-converter.com/) if
      you put your expected number of $AUDIO tokens into the ETH field, it should return the same
      number for “WEI” that you calculated above.
  - After completion, your modal should look like this:

    <img src={useBaseUrl('/img/gnosis-safe/contract-interaction-approve-filled.png')}  />
    <figcaption>caption goes here</figcaption>
  </figure>

  from above, and if all feels right, press “Submit” which will open MetaMask for you to “sign” the
  transaction (likely not yet ready to be submitted depending on number of signers you set up
  above).
- After signing, if you go to the transactions view in the gnosis safe, you should see something
  like this:

  <figcaption>caption goes here</figcaption>
</figure>

- Important note: when the final confirmer hits “confirm”, this will trigger submission of the
  transaction as well. Make sure this address has enough ETH available to proceed.

<figure className="screensnippet-wrapper">

  <figcaption>caption goes here</figcaption>

- Then confirm in MetaMask (we recommend using the “fast” gas setting):

In our case, our gnosis safe is at 0xE0670fBFd3adB450795f5E32820a7dA1399f3AE3 and the transaction is here: https://etherscan.io/tx/0xdf33245469e879c69cd3de4ba78c72741e2f4f617647b70cd53b968c49fd8920
      - Note: the failed transaction in the Gnosis safe history was due to the gas amount being insufficient - MetaMask seems to incorrectly estimate the gas fee for the delegate transaction. This is why there is an added step to edit that included above.

After the transaction confirms, go back to the service that you delegated to and confirm that your address shows up in “Delegators” (you may need to click “View All Delegators”)

      <figure className="screensnippet-wrapper">
        <img src={useBaseUrl('/img/gnosis-safe/audius-dashboard-all-delegators.png')} style={{  }} />
        <figcaption>caption goes here</figcaption>

- You are now successfully delegated!

- To undelegate, you must submit a request to undelegate, allow one week to elapse, then execute
  your request. This requires two function calls via the Gnosis safe that can be used in the flow
  above (select them in the dropdown step when interacting with the delegate manager):
  “requestUndelegateStake” and “undelegateStake”. As mentioned, a sufficient number of blocks must
  pass (46523) between these two function calls.
