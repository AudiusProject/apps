## Use cases and examples

- Plain buttons should be used as a tertiary button.
- Plain buttons are flexible and adaptable to any design. They can be emphasized or hidden.
- Plain buttons are different from text links in that they may contain a leading or trailing icon.

## Do's and don'ts

<ComponentRules
  rules={[
    {
      positive: {
        description:
          'Use plain buttons to reveal or hide additional information.',
        size: 'small',
        component: (
          <PlainButton size='large' iconRight={IconCaretDown}>
            See More
          </PlainButton>
        )

      negative: {
        description:
